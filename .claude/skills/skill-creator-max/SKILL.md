# Skill Creator Max — Agent Skill 対話型生成ツール

## 概要

このスキルは、ユーザーとの **一問一答形式の対話** を通じて、高品質なAgent Skillファイルを生成する。
The Complete Guide to Building Skills for Claude（Anthropic公式完全ガイド）の全項目を網羅し、
YAML frontmatter・ワークフロー設計・参照ファイル構造・テスト計画まで全工程をサポートする。

参照ファイル構成:
- `references/01-yaml-frontmatter.md` — YAMLフロントマター仕様
- `references/02-workflow-patterns.md` — ワークフローパターン集
- `references/03-description-guide.md` — descriptionフィールド記述ガイド
- `references/04-testing-guide.md` — テスト計画ガイド
- `references/05-troubleshooting.md` — トラブルシューティング

---

## CRITICAL: 一問一答ルール

**必ず1つの質問だけを行い、回答を待ってから次に進む。**
複数の質問を同時に行ってはいけない。
ユーザーの回答を受けてから次のステップへ移る。

---

## Phase 1: ユースケース探索

### Q1: 自動化したい作業の確認
```
どんな作業やワークフローをClaudeに覚えさせたいですか？
具体的にどんな操作・手順・判断をClaudeに実行させたいか教えてください。
（例：「毎朝ニュースをまとめてファイルに保存したい」「コードをレビューしてほしい」）
```

回答を受けて内部で分析する：
- スキルのカテゴリ（調査・コード・ファイル操作・Web・API連携など）
- 必要なMCPサーバー（playwright / github / notion / postgresql / slack / なし）
- 入力・出力の形式
- 複雑度（シンプル / 中程度 / 複雑）

---

### Q2: 出力形式の確認
```
スキルの実行結果はどんな形式で欲しいですか？

A. チャットに表示（要約・分析・レポートなど）
B. ファイルに保存（マークダウン・JSON・コードなど）
C. 外部サービスに投稿（GitHub Issue・Notion・Slackなど）
D. A〜Cの組み合わせ

どれが近いですか？（A/B/C/D、または自由記述）
```

---

### Q3: 実行頻度とトリガーの確認
```
このスキルを呼び出すとき、どんな言葉で話しかけますか？
（例：「日経をまとめて」「コードをレビューして」「〇〇を実行して」）

また、どのくらいの頻度で使いますか？
・毎日（定期実行）
・必要なときに随時
```

---

## Phase 2: 技術設計

Q1〜Q3の回答をもとに設計案を提示する。

### 設計案の提示フォーマット
```
【設計案】
━━━━━━━━━━━━━━━━━━━━━━
スキル名    : [name]
トリガー    : 「[word1]」「[word2]」「[word3]」
MCP        : [必要なMCPサーバー / なし]
出力形式    : [チャット / ファイル保存 / 外部連携]
参照ファイル : [あり（references/配下） / なし]

ワークフロー概要:
  Step 1: [処理]
  Step 2: [処理]
  Step 3: [処理]
  ...（完了まで）
━━━━━━━━━━━━━━━━━━━━━━
この設計でよければ「OK」、修正があれば教えてください。
```

---

### Q4: エラー・制約の確認
```
このスキルを使う際に注意が必要なケースはありますか？
（例：ログインが必要・特定フォルダ以外は保存しない・著作権への配慮が必要など）
なければ「特になし」と教えてください。
```

---

### Q5: 参照ファイル分離の確認
```
スキルの判断基準・分類ルール・テンプレートなどを
別ファイル（references/）に分離しますか？

A. 分離する（SKILL.mdを簡潔に保ち、詳細は参照ファイルへ）
   → 複雑なルール・大量の分類基準がある場合に推奨
B. 分離しない（すべてSKILL.mdに記述）
   → シンプルなスキルの場合

どちらにしますか？（A/B）
```

---

## Phase 3: スキルファイル生成

Q1〜Q5の回答を受けて、スキルファイルを生成・保存する。

### 生成ファイル構成
```
.claude/skills/[skill-name]/
├── SKILL.md                  （必須）
└── references/               （Q5でAを選んだ場合）
    ├── [01-content-name].md
    └── [02-content-name].md
```

### SKILL.md生成ルール
詳細は `references/01-yaml-frontmatter.md` を参照。

必須要素：
- YAMLフロントマター（name・description・metadata）
- descriptionに具体的なトリガーワードを含める
- Step番号付きのワークフロー
- エラーハンドリングセクション
- ファイル保存スキルの場合：CRITICALタグ + Writeツール明示

### 参照ファイル生成ルール
詳細は `references/02-workflow-patterns.md` を参照。

- ファイル名: `[番号]-[内容].md`（例: `01-categories.md`）
- SKILL.mdから `references/[filename].md を参照` と明記
- 各ファイルは独立して読めるよう完結した内容にする

### CRITICAL: 必ずWriteツールでファイルを保存すること
生成したSKILL.mdとreferencesファイルは、チャットに表示するだけでなく
**必ずWriteツールを使って `.claude/skills/[name]/` に保存する。**

---

## Phase 4: 検証とテスト計画

スキル生成後、以下のチェックを実行する。

### 自動チェックリスト
詳細は `references/04-testing-guide.md` を参照。

- [ ] YAMLフロントマターが正しく記述されているか
- [ ] descriptionにトリガーワードが含まれているか
- [ ] ワークフローが完結しているか（開始〜終了まで）
- [ ] エラーハンドリングが記述されているか
- [ ] CRITICALタグが適切か（乱用していないか）
- [ ] ファイル保存スキルでWriteツールが明示されているか
- [ ] 参照ファイルへのリンクが正しいか（分離した場合）

### 完了通知フォーマット
```
✅ スキル生成完了

📁 生成ファイル:
  .claude/skills/[name]/SKILL.md
  .claude/skills/[name]/references/[files]（あれば）

🚀 使い方:
  Claude Codeのチャットで「[トリガーワード]」と入力すると起動します。

🧪 推奨テスト手順:
  1. トリガーワードでスキルを呼び出す
  2. 正常系（期待通りの入力）で完走させる
  3. エラーケースを試す

💬 出力を改善したい場合は「スキルを改善して」と入力してください。
```

## エラーハンドリング

### 要件が不明確な場合
具体例を提示して再質問する：
```
もう少し具体的に教えてください。例えば：
・「毎朝GitHubのIssueを確認してSlackに通知したい」
・「コードをコミット前にレビューしてほしい」
・「日経ニュースをまとめてMarkdownに保存したい」
```

### MCPが必要だが未設定の場合
詳細は `references/05-troubleshooting.md` を参照。
```
このスキルには [MCP名] MCPが必要です。
プロジェクトの .mcp.json に以下を追加してから再実行してください：

[設定例を提示]

設定後に「続けて」と入力してください。
```

### 類似スキルが既存の場合
```
似たスキル「[既存スキル名]」が .claude/skills/ に存在します。
・既存スキルを改善しますか？
・新規スキルとして別途作成しますか？
```
