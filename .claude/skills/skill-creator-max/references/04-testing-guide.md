# 04 — テスト計画ガイド

## スキルテストの3段階

```
Stage 1: 静的チェック（ファイル確認）
Stage 2: 正常系テスト（期待通りの動作）
Stage 3: 異常系テスト（エラーケース）
```

---

## Stage 1: 静的チェックリスト

スキルを生成したら、まずファイルを目視で確認する。

### YAMLフロントマター
- [ ] `---` で囲まれているか
- [ ] `name` がフォルダ名と完全一致しているか（kebab-case）
- [ ] `description` にトリガーワードが3個以上含まれているか
- [ ] `metadata.version` が `1.0.0` 形式か
- [ ] 使用するMCPがある場合 `metadata.mcp-server` に記載されているか

### ワークフロー構造
- [ ] Step 番号が連番になっているか（Step 1, 2, 3...）
- [ ] 各Stepに処理内容が具体的に書かれているか
- [ ] 最後のStepで完了通知またはファイル保存が行われているか
- [ ] エラーハンドリングセクションが存在するか

### CRITICALタグ
- [ ] ファイル保存スキルで `Write` ツールが明示されているか
- [ ] CRITICALタグが乱用されていないか（重要箇所のみに使用）

### 参照ファイル（分離した場合）
- [ ] SKILL.md内に `references/[filename].md を参照` と記載されているか
- [ ] 参照先ファイルが実際に存在するか
- [ ] 参照先ファイルが単独で読めるよう完結した内容か

---

## Stage 2: 正常系テスト手順

### テスト手順

1. **トリガーワードでスキルを呼び出す**
   ```
   「[description に記載したトリガーワード]」と入力する
   ```
   → スキルが起動し、最初の処理が始まることを確認

2. **全ステップを通しで実行する**
   - 各Stepが順番に実行されるか
   - 途中で止まらないか
   - エラーなく完走するか

3. **出力を確認する**

   | 出力形式 | 確認内容 |
   |---------|---------|
   | チャット表示 | 期待したフォーマットで表示されているか |
   | ファイル保存 | 正しいパスに正しいファイル名で保存されているか |
   | 外部連携 | APIが正常に呼び出され、データが反映されているか |

### 正常系テストのチェックリスト
- [ ] トリガーワードでスキルが起動した
- [ ] 全Stepが順番に実行された
- [ ] 出力形式が設計通りである
- [ ] ファイル保存の場合、ファイルが存在し内容が正しい
- [ ] 完了通知が表示された

---

## Stage 3: 異常系テスト

### 共通の異常系ケース

| ケース | テスト方法 | 期待する動作 |
|--------|----------|------------|
| MCPが未設定 | `.mcp.json` を削除してから実行 | エラーメッセージと設定手順を表示 |
| 入力ファイルが存在しない | 前提ファイルを削除してから実行 | 「ファイルが見つかりません」と表示して終了 |
| ネットワーク接続なし | オフラインで実行（Web系スキル） | 接続エラーを適切に処理して終了 |
| 権限エラー | 読み取り専用フォルダに保存しようとする | エラーメッセージを表示 |

### スキルタイプ別の異常系

#### ファイル保存スキルの場合
- [ ] 出力先ディレクトリが存在しない場合、自動作成されるか
- [ ] 同名ファイルが既存の場合、上書き or エラーか（設計通りか）

#### Web操作スキルの場合
- [ ] ログインが必要なページにログインなしでアクセスした場合
- [ ] ページ構造が変わって要素が取得できない場合
- [ ] タイムアウトが発生した場合

#### 条件分岐スキルの場合
- [ ] 全条件が一致しない場合（デフォルトケース）の動作
- [ ] 境界値（判定基準の境目）での動作

#### 一問一答スキルの場合
- [ ] 予期しない回答（A/B/Cのどれでもない入力）への対応
- [ ] 空白・無効な入力への対応

## テスト結果の記録

テスト後、以下の形式で結果を記録しておくと改善時に役立つ:

```
## テスト記録 — [スキル名] v[バージョン]
テスト日: YYYY-MM-DD

### 正常系
- [x] トリガー起動: OK
- [x] ワークフロー完走: OK
- [x] ファイル保存: OK（パス: xxx）

### 異常系
- [x] MCP未設定: エラーメッセージ表示 OK
- [ ] ネットワーク切断: 未テスト

### 改善点
- Step 3の処理が遅い（約30秒）→ 並列化を検討
```

## よくある不具合と対処法

### スキルが起動しない

**原因**: descriptionのトリガーワードが実際の発言と一致していない

**対処**:
1. `description` のトリガーワードを確認する
2. ユーザーが使う自然な言葉に修正する
3. `references/03-description-guide.md` を参照して書き直す

---

### ファイルが保存されない

**原因**: WriteツールではなくチャットにMarkdownを表示して終わっている

**対処**:
```markdown
**CRITICAL: 必ずWriteツールでファイルを保存すること。**
チャットに表示するだけでなく、Writeツールを使って以下のパスに保存する:
`C:/path/to/output/filename.md`
```

---

### 途中でスキルが止まる

**原因**: エラーハンドリングが記述されていない

**対処**:
```markdown
## エラーハンドリング

### [エラーケース名]
[エラーが発生した場合の処理]
→「[ユーザーへの通知メッセージ]」を表示してスキルを終了する
```

---

### 参照ファイルが読み込まれない

**原因**: SKILL.md内の参照パスが間違っている

**対処**:
- `references/01-filename.md を参照` という記述がSKILL.mdにあるか確認
- 実際のファイルが `.claude/skills/[name]/references/` に存在するか確認
- ファイル名のスペルミスがないか確認
