# 日経電子版 投資ブリーフィング

## CRITICAL: 実行前の必須確認事項

- Playwright MCPが接続されていること
- 実行前にPlaywrightブラウザで日経電子版にログイン済みであること
- 記事本文の引用（verbatim）は著作権上禁止。必ず自分の言葉でまとめること
- 個別記事の全文を取得してから要約すること（一覧ページのリードのみでの要約は不可）

---

## Step 1: ファイル存在チェック

本日の日付を `YYYY-MM-DD` 形式で取得する。
保存先ディレクトリ: `C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM/`

以下の順でファイルの存在を確認する：

### ケース1: 完全版がすでに存在する
`nikkei-summary-YYYY-MM-DD.md` が存在する場合：
```
「本日（YYYY-MM-DD）の投資ブリーフィング完全版はすでに作成済みです。
ファイル: nikkei-summaries/YYYY-MM/nikkei-summary-YYYY-MM-DD.md」
```
と通知してスキルを終了する。

### ケース2: 朝刊ファイルが存在する
`nikkei-summary-YYYY-MM-DD-morning.md` が存在する場合：
→ **Step 2B（夕刊のみ処理）** へ進む

### ケース3: ファイルが存在しない
→ **Step 2A（通常処理）** へ進む

---

## Step 2A: 通常処理（ファイルなしの場合）

### 2A-1: 朝刊ページにアクセス
Playwrightで `https://www.nikkei.com/paper/` にアクセスし、
当日の朝刊記事一覧を取得する。

### 2A-2: 夕刊の配信状況を確認
夕刊セクションを確認し、当日の夕刊が配信済みかを判定する。

**夕刊が配信済みの場合:**
→ Step 3（朝刊・夕刊の両方を対象に記事収集）へ進む
→ 完了後に `nikkei-summary-YYYY-MM-DD.md` として保存

**夕刊が未配信の場合:**
→ Step 3（朝刊のみ対象に記事収集）へ進む
→ 完了後に `nikkei-summary-YYYY-MM-DD-morning.md` として保存
→ 出力の冒頭に `⚠️ 朝刊のみ（夕刊は配信後に「夕刊を追加して」と入力してください）` と記載する

---

## Step 2B: 夕刊のみ処理（朝刊ファイルが存在する場合）

### 2B-1: 夕刊の配信状況を確認
Playwrightで夕刊セクションにアクセスし、配信済みかを確認する。

**夕刊が未配信の場合:**
```
「夕刊はまだ配信されていません。14〜15時以降に再度実行してください。」
```
と通知してスキルを終了する。

**夕刊が配信済みの場合:**
→ Step 3（夕刊のみ対象に記事収集）へ進む
→ 完了後に既存の朝刊ファイルと統合して `nikkei-summary-YYYY-MM-DD.md` として保存

**CRITICAL: 朝刊ファイルの削除（必ず実行すること）**
完全版ファイルの保存が完了した後、Bash ツールで以下のコマンドを実行して朝刊ファイルを削除する：
```bash
rm "C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM/nikkei-summary-YYYY-MM-DD-morning.md"
```
（YYYY-MM・YYYY-MM-DDは実際の日付に置き換えること）
削除しないと同じ日のファイルが2つ残ってしまうため、必ず削除を実行すること。

---

## Step 3: 投資関連記事の収集とフィルタリング

記事一覧から以下の5カテゴリに該当する記事を選別する。
詳細なカテゴリ基準は `references/article-categories.md` を参照。

**CRITICAL: 記事は多めに収集すること**
「株式投資に少しでも関係しそう」と感じた記事はすべて含める。除外するのは明らかに投資と無関係な記事（芸能・スポーツ・社会面の一般ニュース等）のみ。判断に迷った場合は含める方向で判断する。

### 収集対象カテゴリ
| カテゴリ | 内容 |
|---------|------|
| 🏦 金融政策・マーケット | FRB・日銀・ECBの動向、為替・金利・原油・金、主要株価指数 |
| 📈 個別銘柄・業界 | 決算・M&A・リコール・業界再編など株価に直結するニュース |
| 🌍 地政学リスク | 戦争・選挙・外交摩擦など市場に影響する政治情勢 |
| 📊 経済指標 | GDP・雇用統計・CPI・PMI・小売売上高などの発表と解説 |
| 💡 テクノロジー・産業トレンド | AI・半導体・EV・エネルギーなど中長期で株価に影響するニュース |

### 記事の優先度判定
各記事に投資インパクトを付与する：
- ★★★（高）: 市場全体・為替・金利に直接影響
- ★★（中）: セクター・個別銘柄に影響
- ★（低）: 中長期の参考情報・業界動向・政策動向

★（低）であっても株式投資の参考になる記事はすべて含める。

## Step 4: 個別記事の全文取得と記事コンテンツ生成

### CRITICAL: 必ず個別記事ページを開いて全文を取得すること
記事一覧のリード文のみでの要約は禁止。各記事のURLにアクセスして全文を読んでから記述する。

### CRITICAL: 全記事にリンクを必ず記載すること
`🔗 元記事` の行はすべての記事で必須。URLが取得できない場合も、一覧ページから辿ったURLを記載する。リンクなしで記事を記載することは禁止。

各記事について以下のフォーマットで生成する：

### 記事フォーマット
```
---

### 📌 [記事タイトル]

| 項目 | 内容 |
|------|------|
| 🔗 元記事 | [読む](記事のURL) |
| 📊 投資インパクト | ★★★（高）/ ★★（中）/ ★（低） |
| 📅 掲載 | 朝刊 / 夕刊 |

**概要**
誰が・何を・いつ・なぜ・どうなったか・市場反応を自然な日本語1〜2段落でまとめる。
数値（金利水準、株価変動幅、為替レート、パーセンテージ）は必ず含める。

**詳細**
以下をすべて含めて2〜4段落で記述する：
- 事象が起きた背景・経緯（なぜ今このタイミングか）
- 関係者・当事者の具体的な発言・行動・数値データ
- 他国・他市場・他業界への波及効果
- 今後の見通しや次のイベント（次回会合日程・発表予定など）

> 💡 **投資への示唆**
> → [具体的な示唆1]
> → [具体的な示唆2（あれば）]
> → [注意すべきリスク（あれば）]

📖 **この記事の用語解説**
**[用語・組織名]**: [2〜4文で意味・役割・投資との関係を解説]
**[用語2]**: [解説]
（この記事に金融用語・組織名が登場しない場合はこのセクションを省略する）
```

### 記事コンテンツの品質基準
- 「概要」は1〜2段落、「詳細」は2〜4段落で分けて記述する
- 数値は概要・詳細の両方に含める
- 元記事の言葉をそのまま引用しない（自分の言葉でまとめる）
- 「詳細」には背景・発言・波及効果・今後の見通しを必ず含める
- 投資への示唆はブロック引用（`>`）で視覚的に区別する
- 示唆は記事の内容から合理的に導ける範囲に留める
- 用語解説は記事の直後に記載し、その記事に登場した用語のみ解説する
- 用語の詳細解説テンプレートは `references/terms-glossary.md` を参照

---

## Step 5: 金融用語・組織の解説（各記事直下に記載）

各記事を生成する際、その記事の「投資への示唆」の直後に `📖 この記事の用語解説` セクションを追加する。
詳細な解説テンプレートは `references/terms-glossary.md` を参照。

### 解説対象
- 金融用語: QT・QE・PMI・CPI・FOMC・PER・PBR・ヘッジファンドなど
- 金融組織: FRB・日銀・ECB・IMF・BIS・SEC・CFTC・FSBなど
- その記事に初めて登場した用語のみ解説する（読者が知らない可能性の高い専門用語）
- 同じ用語が複数の記事に登場する場合は、最初の記事でのみ解説する（2回目以降は省略）

### 解説フォーマット（各記事の末尾に追加）
```
📖 **この記事の用語解説**
**[用語・組織名]**: [2〜4文で意味・役割・投資との関係を解説]
**[用語2]**: [解説]
```
用語が登場しない記事はこのセクションを省略する。

### 保存処理

**CRITICAL: 必ず以下のツールを使って保存すること。チャット表示だけで終わらせてはいけない。**

1. **Bash ツールでディレクトリを作成する（存在しない場合）:**
   ```bash
   mkdir -p C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM
   ```

2. **Write ツールでファイルを保存する:**
   - 朝夕両方: `C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM/nikkei-summary-YYYY-MM-DD.md`

3. **ケース2B（夕刊追加）の場合:**
   - Read ツールで既存の morning ファイルの内容を読み込む

     `C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM/nikkei-summary-YYYY-MM-DD.md`

     ```bash
     rm C:/Users/start/Desktop/demo/nikkei-summaries/YYYY-MM/nikkei-summary-YYYY-MM-DD-morning.md
     ```

4. **Write ツールで index.md を更新する:**
   ファイルパス: `C:/Users/start/Desktop/demo/nikkei-summaries/index.md`
   追記行: `| YYYY/M/D | ✅/─ | ✅/─ | [当日の主なトピック3つ] |`

```

「Playwright MCPが接続されていません。
```
